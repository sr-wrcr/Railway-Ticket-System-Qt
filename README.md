# 铁路管理系统（C++ + Qt 图形界面）

> 一个基于 **Qt 框架** 开发的铁路票务桌面应用，提供完整的用户端购票/退票/改签和管理端数据维护功能。

## 🖥️ 项目特色（Qt 图形界面）

本项目**全程采用 Qt 控件**构建用户交互，告别黑框命令行：

- **窗口化操作**：登录窗、注册窗、购票窗、订单管理窗、时刻表窗……全图形界面
- **表格展示**：管理员用 `QTableWidget` 直接编辑数据（列车、站点、时刻表）
- **日历选择器**：购票和查时刻表时用 `QDateEdit` 限制可选日期范围（次日～15天内）
- **消息弹窗**：`QMessageBox` 统一提示错误、警告、成功信息
- **事件驱动**：按钮点击、表格修改、日期切换等全部通过信号/槽响应

## 📌 项目简介

模拟真实铁路票务系统，支持 **普通用户** 和 **系统管理员** 双角色登录。所有数据以文本文件存储，启动时加载，关闭时保存。

## ✨ 主要功能（Qt 界面实现）

### 普通用户
| 功能 | Qt 实现方式 |
|------|-------------|
| 注册 | 表单窗口，实时校验输入（密码强度、手机/身份证格式） |
| 登录 | 下拉选择角色（用户/管理员），验证后跳转不同主窗口 |
| 时刻表查询 | 日历选日期 + 表格展示经停站、到发时间、座位余量和票价 |
| 购票 | 车次下拉框、站点联动选择、动态计算票价并显示 |
| 我的订单 | 表格列出所有订单，支持“退票”和“改签”按钮 |
| 修改个人信息 | 可编辑表单，需验证当前密码 |

### 管理员
| 功能 | Qt 实现方式 |
|------|-------------|
| 用户管理 | 表格展示，只能重置密码（自动设为 `123abc`） |
| 列车/站点/时刻表管理 | 表格直接增删改，实时校验数据格式 |
| 座位/订单查看 | 只读表格，不可编辑 |

## 🧱 技术栈

- **语言**：C++11
- **GUI 框架**：Qt 5.12+（Widgets 模块）
- **设计模式**：分层架构（UI → 业务逻辑 → 数据持久）
- **数据存储**：文本文件（自定义格式，用 `<<` / `>>` 序列化）

## 🚀 如何运行

### 环境要求
- Qt 5.12 或更高版本（需包含 `core` `widgets` 模块）
- 编译器：MinGW 7.3+ / MSVC 2019+ / Clang

### 编译步骤（Qt Creator）
1. 打开项目文件夹中的 `.pro` 文件
2. 选择编译套件（例如 `Desktop Qt 5.15.2 MinGW 64-bit`）
3. 点击 **构建** → **构建项目**
4. 运行生成的可执行文件（建议从 Qt Creator 内运行，或确保 `.txt` 文件在程序所在目录）

> 💡 如果只想体验功能，可以联系开发者获取已编译的 `exe` 及配套数据文件。

## 🧠 核心算法（与 Qt 界面联动）

### 1. 动态票价计算
用户在购票窗口选择 **起点/终点** 后，自动计算票价并显示：
```
票价 = 基础价 × (乘坐站数 / 总站数-1)
```

### 2. 时间冲突检测
购票/改签时，检查用户同一日期是否已有时间重叠的未取消订单（按起终点站实际时间区间比较）。若冲突，弹窗阻止并提示。

### 3. 座位分配与同步
- 购票时从当日可用座位池中分配一个
- 退票/改签后立即释放座位
- 每天自动初始化未来 30 天的座位池

### 4. 数据验证（Qt 实时反馈）
- 输入框失去焦点或点击保存时，立即校验
- 身份证格式、手机号、密码强度、日期范围等
- 校验失败弹窗 `QMessageBox::warning`，不会崩溃

## ⚠️ 异常处理（Qt 弹窗友好提示）

所有错误都通过 `QMessageBox` 呈现，不写晦涩的控制台输出：

| 场景 | 弹窗内容示例 |
|------|-------------|
| 登录用户名不存在 | “用户名不存在” |
| 密码错误 | “密码错误” |
| 购票日期无效 | “请选择第二天到15天内的日期” |
| 时间冲突 | “您已在该时间段内购买了其他车票” |
| 座位售罄 | “座位已售罄” |
| 管理员删除订单 | “订单管理中不支持删除操作” |

此外，底层也使用了 C++ 异常（`std::invalid_argument`、`std::runtime_error`）捕获严重错误，防止程序崩溃。

## 📝 注意事项

1. **管理员账号**：需要预先在 `users.txt` 中设置一条 `role=admin` 的记录，或通过注册后手动修改文件。
2. **数据文件编码**：建议使用 UTF-8 without BOM，避免中文乱码。
3. **Qt 版本兼容**：本项目基于 Qt 5.15 开发，Qt 6 可能需要微调。
4. **改签功能**：改签实际上是“退旧单 + 重新购票”的封装，会重新执行时间冲突检测和座位分配。

## 🎯 总结：为什么用 Qt？

| 对比项 | 控制台版 | Qt 版（本项目） |

| 用户体验 | 键盘输入命令，不直观 | 鼠标点击、表格编辑、日历选择 |

| 错误提示 | 屏幕打印，容易忽略 | 模态弹窗，强制注意 |
| 开发复杂度 | 低 | 中等（需学习信号/槽、布局） |
| 适用场景 | 算法练习、数据结构课设 | 模拟真实桌面软件、软件工程课设 |
